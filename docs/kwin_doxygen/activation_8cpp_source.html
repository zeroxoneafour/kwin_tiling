41/*
42 Prevention of focus stealing:
43
44 KWin tries to prevent unwanted changes of focus, that would result
45 from mapping a new window. Also, some nasty applications may try
46 to force focus change even in cases when ICCCM 4.2.7 doesn't allow it
47 (e.g. they may try to activate their main window because the user
48 definitely "needs" to see something happened - misusing
49 of QWidget::setActiveWindow() may be such case).
50
51 There are 4 ways how a window may become active:
52 - the user changes the active window (e.g. focus follows mouse, clicking
53 on some window's titlebar) - the change of focus will
54 be done by KWin, so there's nothing to solve in this case
55 - the change of active window will be requested using the _NET_ACTIVE_WINDOW
56 message (handled in RootInfo::changeActiveWindow()) - such requests
57 will be obeyed, because this request is meant mainly for e.g. taskbar
58 asking the WM to change the active window as a result of some user action.
59 Normal applications should use this request only rarely in special cases.
60 See also below the discussion of _NET_ACTIVE_WINDOW_TRANSFER.
61 - the change of active window will be done by performing XSetInputFocus()
62 on a window that's not currently active. ICCCM 4.2.7 describes when
63 the application may perform change of input focus. In order to handle
64 misbehaving applications, KWin will try to detect focus changes to
65 windows that don't belong to currently active application, and restore
66 focus back to the currently active window, instead of activating the window
67 that got focus (unfortunately there's no way to FocusChangeRedirect similar
68 to e.g. SubstructureRedirect, so there will be short time when the focus
69 will be changed). The check itself that's done is
70 Workspace::allowWindowActivation() (see below).
71 - a new window will be mapped - this is the most complicated case. If
72 the new window belongs to the currently active application, it may be safely
73 mapped on top and activated. The same if there's no active window,
74 or the active window is the desktop. These checks are done by
75 Workspace::allowWindowActivation().
76 Following checks need to compare times. One time is the timestamp
77 of last user action in the currently active window, the other time is
78 the timestamp of the action that originally caused mapping of the new window
79 (e.g. when the application was started). If the first time is newer than
80 the second one, the window will not be activated, as that indicates
81 futher user actions took place after the action leading to this new
82 mapped window. This check is done by Workspace::allowWindowActivation().
83 There are several ways how to get the timestamp of action that caused
84 the new mapped window (done in X11Window::readUserTimeMapTimestamp()) :
85 - the window may have the _NET_WM_USER_TIME property. This way
86 the application may either explicitly request that the window is not
87 activated (by using 0 timestamp), or the property contains the time
88 of last user action in the application.
89 - KWin itself tries to detect time of last user action in every window,
90 by watching KeyPress and ButtonPress events on windows. This way some
91 events may be missed (if they don't propagate to the toplevel window),
92 but it's good as a fallback for applications that don't provide
93 _NET_WM_USER_TIME, and missing some events may at most lead
94 to unwanted focus stealing.
95 - the timestamp may come from application startup notification.
96 Application startup notification, if it exists for the new mapped window,
97 should include time of the user action that caused it.
98 - if there's no timestamp available, it's checked whether the new window
99 belongs to some already running application - if yes, the timestamp
100 will be 0 (i.e. refuse activation)
101 - if the window is from session restored window, the timestamp will
102 be 0 too, unless this application was the active one at the time
103 when the session was saved, in which case the window will be
104 activated if there wasn't any user interaction since the time
105 KWin was started.
106 - as the last resort, the _KDE_NET_USER_CREATION_TIME timestamp
107 is used. For every toplevel window that is created (see CreateNotify
108 handling), this property is set to the at that time current time.
109 Since at this time it's known that the new window doesn't belong
110 to any existing application (better said, the application doesn't
111 have any other window mapped), it is either the very first window
112 of the application, or it is the only window of the application
113 that was hidden before. The latter case is handled by removing
114 the property from windows before withdrawing them, making
115 the timestamp empty for next mapping of the window. In the sooner
116 case, the timestamp will be used. This helps in case when
117 an application is launched without application startup notification,
118 it creates its mainwindow, and starts its initialization (that
119 may possibly take long time). The timestamp used will be older
120 than any user action done after launching this application.
121 - if no timestamp is found at all, the window is activated.
122 The check whether two windows belong to the same application (same
123 process) is done in X11Window::belongToSameApplication(). Not 100% reliable,
124 but hopefully 99,99% reliable.
125
126 As a somewhat special case, window activation is always enabled when
127 session saving is in progress. When session saving, the session
128 manager allows only one application to interact with the user.
129 Not allowing window activation in such case would result in e.g. dialogs
130 not becoming active, so focus stealing prevention would cause here
131 more harm than good.
132
133 Windows that attempted to become active but KWin prevented this will
134 be marked as demanding user attention. They'll get
135 the _NET_WM_STATE_DEMANDS_ATTENTION state, and the taskbar should mark
136 them specially (blink, etc.). The state will be reset when the window
137 eventually really becomes active.
138
139 There are two more ways how a window can become obtrusive, window stealing
140 focus: By showing above the active window, by either raising itself,
141 or by moving itself on the active desktop.
142 - KWin will refuse raising non-active window above the active one,
143 unless they belong to the same application. Applications shouldn't
144 raise their windows anyway (unless the app wants to raise one
145 of its windows above another of its windows).
146 - KWin activates windows moved to the current desktop (as that seems
147 logical from the user's point of view, after sending the window
148 there directly from KWin, or e.g. using pager). This means
149 applications shouldn't send their windows to another desktop
150 (SELI TODO - but what if they do?)
151
152 Special cases I can think of:
153 - konqueror reusing, i.e. kfmclient tells running Konqueror instance
154 to open new window
155 - without focus stealing prevention - no problem
156 - with ASN (application startup notification) - ASN is forwarded,
157 and because it's newer than the instance's user timestamp,
158 it takes precedence
159 - without ASN - user timestamp needs to be reset, otherwise it would
160 be used, and it's old; moreover this new window mustn't be detected
161 as window belonging to already running application, or it wouldn't
162 be activated - see X11Window::sameAppWindowRoleMatch() for the (rather ugly)
163 hack
164 - konqueror preloading, i.e. window is created in advance, and kfmclient
165 tells this Konqueror instance to show it later
166 - without focus stealing prevention - no problem
167 - with ASN - ASN is forwarded, and because it's newer than the instance's
168 user timestamp, it takes precedence
169 - without ASN - user timestamp needs to be reset, otherwise it would
170 be used, and it's old; also, creation timestamp is changed to
171 the time the instance starts (re-)initializing the window,
172 this ensures creation timestamp will still work somewhat even in this case
173 - KUniqueApplication - when the window is already visible, and the new instance
174 wants it to activate
175 - without focus stealing prevention - _NET_ACTIVE_WINDOW - no problem
176 - with ASN - ASN is forwarded, and set on the already visible window, KWin
177 treats the window as new with that ASN
178 - without ASN - _NET_ACTIVE_WINDOW as application request is used,
179 and there's no really usable timestamp, only timestamp
180 from the time the (new) application instance was started,
181 so KWin will activate the window *sigh*
182 - the bad thing here is that there's absolutely no chance to recognize
183 the case of starting this KUniqueApp from Konsole (and thus wanting
184 the already visible window to become active) from the case
185 when something started this KUniqueApp without ASN (in which case
186 the already visible window shouldn't become active)
187 - the only solution is using ASN for starting applications, at least silent
188 (i.e. without feedback)
189 - when one application wants to activate another application's window (e.g. KMail
190 activating already running KAddressBook window ?)
191 - without focus stealing prevention - _NET_ACTIVE_WINDOW - no problem
192 - with ASN - can't be here, it's the KUniqueApp case then
193 - without ASN - _NET_ACTIVE_WINDOW as application request should be used,
194 KWin will activate the new window depending on the timestamp and
195 whether it belongs to the currently active application
196
197 _NET_ACTIVE_WINDOW usage:
198 data.l[0]= 1 ->app request
199 = 2 ->pager request
200 = 0 - backwards compatibility
201 data.l[1]= timestamp
202*/

216void Workspace::setActiveWindow(Window *window)
217{
218 if (m_activeWindow == window) {
219 return;
220 }
221
222 if (active_popup && m_activePopupWindow != window && m_setActiveWindowRecursion == 0) {
223 closeActivePopup();
224 }
225 if (m_userActionsMenu->hasWindow() && !m_userActionsMenu->isMenuWindow(window) && m_setActiveWindowRecursion == 0) {
226 m_userActionsMenu->close();
227 }
228 StackingUpdatesBlocker blocker(this);
229 ++m_setActiveWindowRecursion;
230 updateFocusMousePosition(Cursors::self()->mouse()->pos());
231 if (m_activeWindow != nullptr) {
232 // note that this may call setActiveWindow( NULL ), therefore the recursion counter
233 m_activeWindow->setActive(false);
234 }
235 m_activeWindow = window;
236 Q_ASSERT(window == nullptr || window->isActive());
237
238 if (m_activeWindow) {
239 m_lastActiveWindow = m_activeWindow;
240 m_focusChain->update(m_activeWindow, FocusChain::MakeFirst);
241 m_activeWindow->demandAttention(false);
242
243 // activating a client can cause a non active fullscreen window to loose the ActiveLayer status on > 1 screens
244 if (outputs().count() > 1) {
245 for (auto it = m_allClients.begin(); it != m_allClients.end(); ++it) {
246 if (*it != m_activeWindow && (*it)->layer() == ActiveLayer && (*it)->output() == m_activeWindow->output()) {
247 (*it)->updateLayer();
248 }
249 }
250 }
251 }
252
253 updateToolWindows(false);
254 if (window) {
255 disableGlobalShortcutsForClient(window->rules()->checkDisableGlobalShortcuts(false));
256 } else {
257 disableGlobalShortcutsForClient(false);
258 }
259
260 updateStackingOrder(); // e.g. fullscreens have different layer when active/not-active
261
262 if (rootInfo()) {
263 rootInfo()->setActiveClient(m_activeWindow);
264 }
265
266 Q_EMIT windowActivated(m_activeWindow);
267 --m_setActiveWindowRecursion;
268}

282void Workspace::activateWindow(Window *window, bool force)
283{
284 if (window == nullptr) {
285 focusToNull();
286 setActiveWindow(nullptr);
287 return;
288 }
289 raiseWindow(window);
290 if (!window->isOnCurrentDesktop()) {
291 ++block_focus;
292 switch (options->activationDesktopPolicy()) {
293 case Options::ActivationDesktopPolicy::SwitchToOtherDesktop:
294 VirtualDesktopManager::self()->setCurrent(window->desktops().constLast());
295 break;
296 case Options::ActivationDesktopPolicy::BringToCurrentDesktop:
297 window->enterDesktop(VirtualDesktopManager::self()->currentDesktop());
298 break;
299 }
300 --block_focus;
301 }
302#if KWIN_BUILD_ACTIVITIES
303 if (!window->isOnCurrentActivity()) {
304 ++block_focus;
305 // DBUS!
306 // first isn't necessarily best, but it's easiest
307 m_activities->setCurrent(window->activities().constFirst());
308 --block_focus;
309 }
310#endif
311 if (window->isMinimized()) {
312 window->unminimize();
313 }
314
315 // ensure the window is really visible - could eg. be a hidden utility window, see bug #348083
316 window->showClient();
317
318 // TODO force should perhaps allow this only if the window already contains the mouse
319 if (options->focusPolicyIsReasonable() || force) {
320 requestFocus(window, force);
321 }
322
323 // Don't update user time for windows that have focus stealing workaround.
324 // As they usually belong to the current active window but fail to provide
325 // this information, updating their user time would make the user time
326 // of the currently active window old, and reject further activation for it.
327 // E.g. typing URL in minicli which will show kio_uiserver dialog (with workaround),
328 // and then kdesktop shows dialog about SSL certificate.
329 // This needs also avoiding user creation time in X11Window::readUserTimeMapTimestamp().
330 if (X11Window *x11Window = dynamic_cast<X11Window *>(window)) {
331 // updateUserTime is X11 specific
332 x11Window->updateUserTime();
333 }
334}

348bool Workspace::takeActivity(Window *window, ActivityFlags flags)
349{
350 // the 'if ( window == m_activeWindow ) return;' optimization mustn't be done here
351 if (!focusChangeEnabled() && (window != m_activeWindow)) {
352 flags &= ~ActivityFocus;
353 }
354
355 if (!window) {
356 focusToNull();
357 return true;
358 }
359
360 if (flags & ActivityFocus) {
361 Window *modal = window->findModal();
362 if (modal != nullptr && modal != window) {
363 if (modal->desktops() != window->desktops()) {
364 modal->setDesktops(window->desktops());
365 }
366 if (!modal->isShown() && !modal->isMinimized()) { // forced desktop or utility window
367 activateWindow(modal); // activating a minimized blocked window will unminimize its modal implicitly
368 }
369 // if the click was inside the window (i.e. handled is set),
370 // but it has a modal, there's no need to use handled mode, because
371 // the modal doesn't get the click anyway
372 // raising of the original window needs to be still done
373 if (flags & ActivityRaise) {
374 raiseWindow(window);
375 }
376 window = modal;
377 }
378 cancelDelayFocus();
379 }
380 if (!flags.testFlag(ActivityFocusForce) && (window->isDock() || window->isSplash())) {
381 // toplevel menus and dock windows don't take focus if not forced
382 // and don't have a flag that they take focus
383 if (!window->dockWantsInput()) {
384 flags &= ~ActivityFocus;
385 }
386 }
387 if (window->isShade()) {
388 if (window->wantsInput() && (flags & ActivityFocus)) {
389 // window cannot accept focus, but at least the window should be active (window menu, et. al. )
390 window->setActive(true);
391 focusToNull();
392 }
393 flags &= ~ActivityFocus;
394 }
395 if (!window->isShown()) { // shouldn't happen, call activateWindow() if needed
396 qCWarning(KWIN_CORE) << "takeActivity: not shown";
397 return false;
398 }
399
400 bool ret = true;
401
402 if (flags & ActivityFocus) {
403 ret &= window->takeFocus();
404 }
405 if (flags & ActivityRaise) {
406 workspace()->raiseWindow(window);
407 }
408
409 if (!window->isOnActiveOutput()) {
410 setActiveOutput(window->output());
411 }
412
413 return ret;
414}

429Window *Workspace::windowUnderMouse(Output *output) const
430{
431 auto it = stackingOrder().constEnd();
432 while (it != stackingOrder().constBegin()) {
433 auto window = *(--it);
434 if (!window->isClient()) {
435 continue;
436 }
437
438 // rule out windows which are not really visible.
439 // the screen test is rather superfluous for xrandr & twinview since the geometry would differ -> TODO: might be dropped
440 if (!(window->isShown() && window->isOnCurrentDesktop() && window->isOnCurrentActivity() && window->isOnOutput(output) && !window->isShade())) {
441 continue;
442 }
443
444 if (exclusiveContains(window->frameGeometry(), Cursors::self()->mouse()->pos())) {
445 return window;
446 }
447 }
448 return nullptr;
449}

451// deactivates 'window' and activates next window
452bool Workspace::activateNextWindow(Window *window)
453{
454 // if 'c' is not the active or the to-become active one, do nothing
455 if (!(window == m_activeWindow || (should_get_focus.count() > 0 && window == should_get_focus.last()))) {
456 return false;
457 }
458
459 closeActivePopup();
460
461 if (window != nullptr) {
462 if (window == m_activeWindow) {
463 setActiveWindow(nullptr);
464 }
465 should_get_focus.removeAll(window);
466 }
467
468 // if blocking focus, move focus to the desktop later if needed
469 // in order to avoid flickering
470 if (!focusChangeEnabled()) {
471 focusToNull();
472 return true;
473 }
474
475 if (!options->focusPolicyIsReasonable()) {
476 return false;
477 }
478
479 Window *focusCandidate = nullptr;
480
481 VirtualDesktop *desktop = VirtualDesktopManager::self()->currentDesktop();
482
483 if (!focusCandidate && showingDesktop()) {
484 focusCandidate = findDesktop(true, desktop); // to not break the state
485 }
486
487 if (!focusCandidate && options->isNextFocusPrefersMouse()) {
488 focusCandidate = windowUnderMouse(window ? window->output() : workspace()->activeOutput());
489 if (focusCandidate && (focusCandidate == window || focusCandidate->isDesktop())) {
490 // should rather not happen, but it cannot get the focus. rest of usability is tested above
491 focusCandidate = nullptr;
492 }
493 }
494
495 if (!focusCandidate) { // no suitable window under the mouse -> find sth. else
496 // first try to pass the focus to the (former) active clients leader
497 if (window && window->isTransient()) {
498 auto leaders = window->mainWindows();
499 if (leaders.count() == 1 && m_focusChain->isUsableFocusCandidate(leaders.at(0), window)) {
500 focusCandidate = leaders.at(0);
501 raiseWindow(focusCandidate); // also raise - we don't know where it came from
502 }
503 }
504 if (!focusCandidate) {
505 // nope, ask the focus chain for the next candidate
506 focusCandidate = m_focusChain->nextForDesktop(window, desktop);
507 }
508 }
509
510 if (focusCandidate == nullptr) { // last chance: focus the desktop
511 focusCandidate = findDesktop(true, desktop);
512 }
513
514 if (focusCandidate != nullptr) {
515 requestFocus(focusCandidate);
516 } else {
517 focusToNull();
518 }
519
520 return true;
521}

523void Workspace::switchToOutput(Output *output)
524{
525 if (!options->focusPolicyIsReasonable()) {
526 return;
527 }
528 closeActivePopup();
529 VirtualDesktop *desktop = VirtualDesktopManager::self()->currentDesktop();
530 Window *get_focus = m_focusChain->getForActivation(desktop, output);
531 if (get_focus == nullptr) {
532 get_focus = findDesktop(true, desktop);
533 }
534 if (get_focus != nullptr && get_focus != mostRecentlyActivatedWindow()) {
535 requestFocus(get_focus);
536 }
537 setActiveOutput(output);
538}

558// basically the same like allowWindowActivation(), this time allowing
559// a window to be fully raised upon its own request (XRaiseWindow),
560// if refused, it will be raised only on top of windows belonging
561// to the same application
562bool Workspace::allowFullClientRaising(const KWin::Window *window, xcb_timestamp_t time)
563{
564 int level = window->rules()->checkFSP(options->focusStealingPreventionLevel());
565 if (sessionManager()->state() == SessionState::Saving && level <= 2) { // <= normal
566 return true;
567 }
568 Window *ac = mostRecentlyActivatedWindow();
569 if (level == 0) { // none
570 return true;
571 }
572 if (level == 4) { // extreme
573 return false;
574 }
575 if (ac == nullptr || ac->isDesktop()) {
576 qCDebug(KWIN_CORE) << "Raising: No window active, allowing";
577 return true; // no active window -> always allow
578 }
579 // TODO window urgency -> return true?
580 if (Window::belongToSameApplication(window, ac, Window::SameApplicationCheck::RelaxedForActive)) {
581 qCDebug(KWIN_CORE) << "Raising: Belongs to active application";
582 return true;
583 }
584 if (level == 3) { // high
585 return false;
586 }
587 xcb_timestamp_t user_time = ac->userTime();
588 qCDebug(KWIN_CORE) << "Raising, compared:" << time << ":" << user_time
589 << ":" << (NET::timestampCompare(time, user_time) >= 0);
590 return NET::timestampCompare(time, user_time) >= 0; // time >= user_time
591}

599bool Workspace::restoreFocus()
600{
601 // this updateXTime() is necessary - as FocusIn events don't have
602 // a timestamp *sigh*, kwin's timestamp would be older than the timestamp
603 // that was used by whoever caused the focus change, and therefore
604 // the attempt to restore the focus would fail due to old timestamp
605 kwinApp()->updateXTime();
606 if (should_get_focus.count() > 0) {
607 return requestFocus(should_get_focus.last());
608 } else if (m_lastActiveWindow) {
609 return requestFocus(m_lastActiveWindow);
610 }
611 return true;
612}

657xcb_timestamp_t X11Window::readUserTimeMapTimestamp(const KStartupInfoId *asn_id, const KStartupInfoData *asn_data,
658 bool session) const
659{
660 xcb_timestamp_t time = info->userTime();
661 // qDebug() << "User timestamp, initial:" << time;
662 //^^ this deadlocks kwin --replace sometimes.
663
664 // newer ASN timestamp always replaces user timestamp, unless user timestamp is 0
665 // helps e.g. with konqy reusing
666 if (asn_data != nullptr && time != 0) {
667 if (asn_id->timestamp() != 0
668 && (time == -1U || NET::timestampCompare(asn_id->timestamp(), time) > 0)) {
669 time = asn_id->timestamp();
670 }
671 }
672 qCDebug(KWIN_CORE) << "User timestamp, ASN:" << time;
673 if (time == -1U) {
674 // The window doesn't have any timestamp.
675 // If it's the first window for its application
676 // (i.e. there's no other window from the same app),
677 // use the _KDE_NET_WM_USER_CREATION_TIME trick.
678 // Otherwise, refuse activation of a window
679 // from already running application if this application
680 // is not the active one (unless focus stealing prevention is turned off).
681 X11Window *act = dynamic_cast<X11Window *>(workspace()->mostRecentlyActivatedWindow());
682 if (act != nullptr && !belongToSameApplication(act, this, SameApplicationCheck::RelaxedForActive)) {
683 bool first_window = true;
684 auto sameApplicationActiveHackPredicate = [this](const X11Window *cl) {
685 // ignore already existing splashes, toolbars, utilities and menus,
686 // as the app may show those before the main window
687 return !cl->isSplash() && !cl->isToolbar() && !cl->isUtility() && !cl->isMenu()
688 && cl != this && X11Window::belongToSameApplication(cl, this, SameApplicationCheck::RelaxedForActive);
689 };
690 if (isTransient()) {
691 auto clientMainClients = [this]() {
692 QList<X11Window *> ret;
693 const auto mcs = mainWindows();
694 for (auto mc : mcs) {
695 if (X11Window *c = dynamic_cast<X11Window *>(mc)) {
696 ret << c;
697 }
698 }
699 return ret;
700 };
701 if (act->hasTransient(this, true)) {
702 ; // is transient for currently active window, even though it's not
703 // the same app (e.g. kcookiejar dialog) -> allow activation
704 } else if (groupTransient() && findInList<X11Window, X11Window>(clientMainClients(), sameApplicationActiveHackPredicate) == nullptr) {
705 ; // standalone transient
706 } else {
707 first_window = false;
708 }
709 } else {
710 if (workspace()->findClient(sameApplicationActiveHackPredicate)) {
711 first_window = false;
712 }
713 }
714 // don't refuse if focus stealing prevention is turned off
715 if (!first_window && rules()->checkFSP(options->focusStealingPreventionLevel()) > 0) {
716 qCDebug(KWIN_CORE) << "User timestamp, already exists:" << 0;
717 return 0; // refuse activation
718 }
719 }
720 // Creation time would just mess things up during session startup,
721 // as possibly many apps are started up at the same time.
722 // If there's no active window yet, no timestamp will be needed,
723 // as plain Workspace::allowWindowActivation() will return true
724 // in such case. And if there's already active window,
725 // it's better not to activate the new one.
726 // Unless it was the active window at the time
727 // of session saving and there was no user interaction yet,
728 // this check will be done in manage().
729 if (session) {
730 return -1U;
731 }
732 time = readUserCreationTime();
733 }
734 qCDebug(KWIN_CORE) << "User timestamp, final:" << this << ":" << time;
735 return time;
736}

810// focus_in -> the window got FocusIn event
811// ignore_desktop - call comes from _NET_ACTIVE_WINDOW message, don't refuse just because of window
812// is on a different desktop
813bool X11Window::allowWindowActivation(xcb_timestamp_t time, bool focus_in, bool ignore_desktop)
814{
815 auto window = this;
816 // options->focusStealingPreventionLevel :
817 // 0 - none - old KWin behaviour, new windows always get focus
818 // 1 - low - focus stealing prevention is applied normally, when unsure, activation is allowed
819 // 2 - normal - focus stealing prevention is applied normally, when unsure, activation is not allowed,
820 // this is the default
821 // 3 - high - new window gets focus only if it belongs to the active application,
822 // or when no window is currently active
823 // 4 - extreme - no window gets focus without user intervention
824 if (time == -1U) {
825 time = window->userTime();
826 }
827 const FSP::Level level = (FSP::Level)window->rules()->checkFSP(options->focusStealingPreventionLevel());
828 if (workspace()->sessionManager()->state() == SessionState::Saving && level <= FSP::Medium) { // <= normal
829 return true;
830 }
831 Window *ac = workspace()->mostRecentlyActivatedWindow();
832 if (focus_in) {
833 if (workspace()->inShouldGetFocus(window)) {
834 return true; // FocusIn was result of KWin's action
835 }
836 // Before getting FocusIn, the active Client already
837 // got FocusOut, and therefore got deactivated.
838 ac = workspace()->lastActiveWindow();
839 }
840 if (time == 0) { // explicitly asked not to get focus
841 if (!window->rules()->checkAcceptFocus(false)) {
842 return false;
843 }
844 }
845 const FSP::Level protection = (FSP::Level)(ac ? ac->rules()->checkFPP(2) : FSP::None);
846
847 // stealing is unconditionally allowed (NETWM behavior)
848 if (level == FSP::None || protection == FSP::None) {
849 return true;
850 }
851
852 // The active window "grabs" the focus or stealing is generally forbidden
853 if (level == FSP::Extreme || protection == FSP::Extreme) {
854 return false;
855 }
856
857 // Desktop switching is only allowed in the "no protection" case
858 if (!ignore_desktop && !window->isOnCurrentDesktop()) {
859 return false; // allow only with level == 0
860 }
861
862 // No active window, it's ok to pass focus
863 // NOTICE that extreme protection needs to be handled before to allow protection on unmanged windows
864 if (ac == nullptr || ac->isDesktop()) {
865 qCDebug(KWIN_CORE) << "Activation: No window active, allowing";
866 return true; // no active window -> always allow
867 }
868
869 // TODO window urgency -> return true?
870
871 // Unconditionally allow intra-window passing around for lower stealing protections
872 // unless the active window has High interest
873 if (Window::belongToSameApplication(window, ac, Window::SameApplicationCheck::RelaxedForActive) && protection < FSP::High) {
874 qCDebug(KWIN_CORE) << "Activation: Belongs to active application";
875 return true;
876 }
877
878 if (!window->isOnCurrentDesktop()) { // we allowed explicit self-activation across virtual desktops
879 return false; // inside a window or if no window was active, but not otherwise
880 }
881
882 // High FPS, not intr-window change. Only allow if the active window has only minor interest
883 if (level > FSP::Medium && protection > FSP::Low) {
884 return false;
885 }
886
887 if (time == -1U) { // no time known
888 qCDebug(KWIN_CORE) << "Activation: No timestamp at all";
889 // Only allow for Low protection unless active window has High interest in focus
890 if (level < FSP::Medium && protection < FSP::High) {
891 return true;
892 }
893 // no timestamp at all, don't activate - because there's also creation timestamp
894 // done on CreateNotify, this case should happen only in case application
895 // maps again already used window, i.e. this won't happen after app startup
896 return false;
897 }
898
899 // Low or medium FSP, usertime comparism is possible
900 const xcb_timestamp_t user_time = ac->userTime();
901 qCDebug(KWIN_CORE) << "Activation, compared:" << window << ":" << time << ":" << user_time
902 << ":" << (NET::timestampCompare(time, user_time) >= 0);
903 return NET::timestampCompare(time, user_time) >= 0; // time >= user_time
904}
