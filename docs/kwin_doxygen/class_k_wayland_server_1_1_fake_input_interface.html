Detailed Description

Represents the Global for org_kde_kwin_fake_input interface.

The fake input interface allows clients to send fake input events to the Wayland server. For the actual events it creates a FakeInputDevice. Whenever the FakeInputInterface creates a device the signal deviceCreated gets emitted.

Accepting fake input events is a security risk. The server should make a dedicated decision about whether it wants to accept fake input events from a device. Because of that by default no events are forwarded to the server. The device needs to request authentication and the server must explicitly authenticate the device. The recommendation is that the server only accepts input for in some way trusted clients.
